Криптографическая защита целостности информации

Шифрование

Использование ключа шифрования позволяет защитить информацию от несанкционированного доступа извне. Тип используемого шифрования играет важную роль в защите целостности информации. Существует два основных типа шифрования: симметричное и несимметричное.

Ниже будут рассмотрены оба типа, их плюсы и минусы, а также приведены примеры.

Симметричное шифрование

Наиболее распространенный тип шифрования – это симметричный тип, в литературе его также называют традиционным шифрованием [2]. Долгое время этот тип являлся единственным, так продолжалось до изобретения шифрования с открытым ключом. Симметричный тип шифрования заключается в том, что используется один и тот же ключ, как для шифрования данных, так и для их расшифровки, при этом ключ является секретным. Информация шифруется небольшими блоками, размер которых, как правило, кратен 32 битам.

Плюсы:

  • Высокая скорость: симметричные алгоритмы работают намного быстрее – именно поэтому они часто используются для шифрования больших объемов данных [2].
  • Простота реализации: симметричные алгоритмы проще в реализации и требуют меньших вычислительных ресурсов [2].

Минусы:

  • Управление: требуется безопасное управление ключами, поэтому их передача и хранение требуют особой осторожности.

Примерами симметричных алгоритмов являются AES,  3DES, IDEA и др. [4].

Ярким и простым примером симметричного алгоритма является шифр Цезаря, который будет рассмотрен ниже. Его суть состоит в том, что каждый символ в последовательности заменяется другим, отстоящим от него на фиксированное число позиций. Он выражается формулами 1.1 и 1.2 модульной арифметики:

         (1.1)

        (1.2)

где x – символ исходного текста, y – символ зашифрованного текста, n – мощность алфавита, k – ключ.

Ниже будет приведен пример реализации алгоритмы на Python.

from typing import Union, Literal, List
from pydantic import BaseModel

# заводим словари для латиницы и кириллицы
alphabet: List[str] = ['a', 'b', 'c', 'd', 'e', 'f', 'g',    'h', 'i', 'j','k', 'l', 'm', 'n', 'o', 'p', 'q', 'r', 's', 't', 'u', 'v', 'w', 'x', 'y', 'z']
alphabet_rus: List[str] = ['а', 'б', 'в', 'г', 'д', 'е', 'ё', 'ж', 'з', 'и', 'й', 'к', 'л', 'м', 'н', 'о', 'п', 'р', 'с', 'т', 'у', 'ф', 'х', 'ц', 'ч', 'ш', 'щ', 'ъ', 'ы', 'ь', 'э', 'ю', 'я']

# используем pydantic для валидации данных, не влияет на сам алгоритм

class Parametrs(BaseModel):
  key: int
  language: Union[Literal["eng"], Literal["rus"]]
  mode: Union[Literal["coder"], Literal["decoder"]]

def ceaser(key: int, text: str, lang: str, mode: str = "coder") -> str:
  res: str = ""
  new_letter: str = ""

  p = Parametrs(key=key, language=lang, mode=mode)

  res: str = ""
  new_letter: str = ""
  up: bool = False  # проверка на заглавные буквы
  # будет происходит шифрование или дешифрование строки
  m: int = -1 if p.mode == "decoder" else 1
  # какой алфавит используется в исходной строке
  alph: List[str] = alphabet_rus if p.language == "rus" else alphabet
  key = p.key  # ключ шифрования

  for letter in text:
      if letter.isupper():
          up = True

      letter = letter.lower()
      new_letter = alph[(alph.index(letter) + key * m) %
                        len(alph)] if letter in alph else letter
      new_letter = new_letter.upper() if up else new_letter
      res += new_letter
      up = False
  return res

Асимметричное шифрование

Ассиметричное шифрование заключается в том, что можно использовать разные алгоритмы для шифрования и расшифрования. Если в симметричном шифровании используется только один ключ, здесь используется два ключа, связанных между собой - один из ключей является открытым – и используется для шифрования, другой является секретным - и используется для расшифровки. Только конкретный пользователь или устройство знает секретный (закрытый) ключ, в то время как открытый ключ распространяется среди всех пользователей или устройств, участвующих в обмене данными. [7] Примерами ассиметричных алгоритмов являются: RSA, OAEP.

Плюсы:

  • Безопасность: ассиметричное шифрование позволяет безопасно обмениваться ключами через незащищенные каналы. Открытые ключи публикуются, а закрытые хранятся в секрете.
  • Аутентификация: позволяет проверять подлинность отправителя сообщения.

Минусы:

  • Низкая скорость: основным недостатком асимметричных алгоритмов является скорость, они намного медленнее, чем симметричные алгоритмы.

Хеширование

Что такое хеширование? Для нужно обратиться к понятию “контрольные суммы” – это функции, принимающие любые входные данные и на выходе выдающие число фиксированной длины. При изменении даже малой части входных данных, контрольная сумма будет вычислена иначе. Хеш-функции – это специальные функции контрольной суммы, при которых невозможно узнать входные данные. Вывод хеш-функций называют хешем.

Рис. 5  Хеширование простого файла. Эта хеш‑функция выдает двузначное число с 256 битами в качестве выходных данных, представленное в шестнадцатеричном формате. Невозможно найти другие входные данные, которые произведут тот же результат. [1]

Для многих языков программирования существуют специальные библиотеки, в которых реализованы алгоритмы безопасного хеширование. Так, для python можно выделить hashlib [9].

У хеш-функций множество применений.  Так,  они могут использоваться для защиты от фальсификации передаваемой информации – при сравнении хешей, снятых до и после отправки данных, любое изменение можно увидеть – даже при изменении всего на 1 байт, хеши будут отличаться. Наиболее распространенный способ применения хеш-функций – хранение паролей. Онлайн-сервисы хранят хеши паролей, и не знают, как в действительности выглядит пароль пользователя. Антивирусы тоже используют хеширование для обнаружения вредоносного ПО, храня кеши какого-либо узнаваемого компонента.
